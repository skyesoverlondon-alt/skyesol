Security posture (the honest version)

Skye Intake Vault uses a passcode gate. The passcode is not stored as plain text; it’s stored as a salted hash in browser storage. For best protection, pair the vault with device-level security (screen lock, biometrics, encrypted storage).

Simple policy statement you can publish:
“Vault access is protected by a local passcode gate. Records autosave locally and can be exported for backup. Use device security for best protection.”

Install Skye Intake Vault Offline (PWA)

Installing as a PWA makes Skye Intake Vault feel like a native app and keeps it usable when your connection drops.

1 Desktop (Chrome)
Open skyeintakevault.netlify.app → look for the install icon in the address bar (or menu → Install app) → confirm install.
2 Android (Chrome)
Menu (⋮) → Add to Home screen or Install app → confirm.
3 iPhone/iPad (Safari)
Share icon → Add to Home Screen → Add.
4 Field workflow
Install → set vault passcode → run intake offline → attach proof → export JSON backup when convenient.
Tip: For best protection, keep your device locked (PIN/biometric) and treat the vault passcode like a key. The vault is designed to be fast and local—device security completes the armor.
