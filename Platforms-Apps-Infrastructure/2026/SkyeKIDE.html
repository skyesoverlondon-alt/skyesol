The Problem

Local dev environments are a liability

You spend half your setup time managing Node versions, VS Code extensions, and conflicting dependencies. Then your machine dies. Or you're on a different computer. Or a client hands you their laptop. The code tool vanishes. The AI stays behind.

⚙️

Setup Overhead

Node version managers, package conflicts, extension packs for every language — thirty minutes of friction before you write one line.

🔌

Device Lock-In

Your environment is on one machine. Move to another and you start from scratch. Cloud tools exist but they're heavy, slow, and priced for enterprise teams.

🤖

AI Bolted On

Most browser code editors have no AI layer. The ones that do charge $20/month subscriptions to the same model you already have access to through Kaixu.

🚀

Deploy Gap

Edit-to-deploy requires switching tabs, logging in somewhere else, and triggering a CI run. In a browser IDE that should be one button.

How It Works

Open → Code → AI → Ship

Four steps from idea to live deploy. No terminals open on the side.

01
Open or Import

Start a new project or import a ZIP

Create a blank project and add files, or drag any ZIP archive onto the file tree to instantly unpack an existing codebase. SkyeKIDE restores your last session automatically on every load.

02
Write & Refactor

Edit with full syntax support and live preview

Multi-language editor with bracket matching, auto-indent, and search/replace across all files. Toggle the live preview panel to see HTML/CSS changes render in real time as you type.

03
AI Assist

Prompt kAIxU to complete, explain, or refactor

Select a function and ask kAIxU to add error handling. Describe a feature and get a patch back. The AI panel routes through your Kaixu virtual key — your budget, controlled by spend cap, billed per request.

04
Deploy

Push to Netlify in one click via KaixuPush

Hit Deploy. SkyeKIDE pipes your file set to the KaixuPush endpoint — digests only changed files, uploads through the gateway, and returns a live Netlify URL. Metered, logged, and invoiced through the SOL gateway.
